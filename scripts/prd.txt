# 개요  
ROUTE DASHBOARD는 사용자가 보기 쉽고 정돈된 표 형태로 항목들을 확인할 수 있는 대시보드입니다. 사용자가 항목(row)을 클릭하면 하단에 확장 섹션이 열려 상세 정보를 시각적으로 제공합니다. 주요 사용자는 데이터 기반 관리가 필요한 실무자 및 관리자이며, CSV 기반 자동 데이터 로딩, 이미지/코멘트 첨부, 강력한 필터링, 다크모드 등 직관적이고 현대적인 UX를 제공합니다.

# 주요 기능  
- **CSV 자동 로드**  
  - 앱 실행 시 data/ 폴더 내 CSV 파일을 자동으로 로드하여 표 형식으로 렌더링합니다.  
  - 사용자는 별도의 입력 없이 최신 데이터를 바로 확인할 수 있습니다.

- **Row 클릭 시 확장 디테일 뷰**  
  - 각 row를 클릭하면 하단에 최대 10개의 상세 정보가 자연스러운 애니메이션과 함께 표시됩니다.  
  - 상세 정보에는 첨부된 이미지와 코멘트가 포함될 수 있습니다.

- **Row별 이미지/코멘트 첨부**  
  - 각 row에 이미지 파일 업로드 및 코멘트 입력이 가능합니다.  
  - 표에는 📷 이미지 개수와 💬 코멘트 개수만 아이콘과 숫자로 표시됩니다.

- **필터 기능**  
  - FAB, DEVICE, TECH, 제목 등 다양한 기준으로 row를 필터링할 수 있습니다.  
  - 드롭다운 또는 버튼 그룹 형태로 제공됩니다.

- **다크모드 지원**  
  - Tailwind CSS의 dark: 유틸리티를 활용합니다.  
  - 시스템 설정에 따라 자동 적용되며, 수동 토글 옵션도 제공합니다.

# 사용자 경험  
- **주요 사용자**: 데이터 관리 실무자, 관리자  
- **주요 플로우**:  
  1. 앱 접속 → CSV 데이터 자동 로드  
  2. 필터로 원하는 항목 추출  
  3. row 클릭 → 상세 정보 및 첨부 이미지/코멘트 확인  
  4. 필요시 이미지/코멘트 추가  
- **UI/UX 고려사항**:  
  - 고정 헤더와 스크롤 가능한 표  
  - 확장 섹션은 자연스러운 애니메이션 적용  
  - 첨부 개수는 아이콘+숫자(📷 3 | 💬 1)로만 표기  
  - 다크모드 자동/수동 전환 지원

# 기술 아키텍처  
- **프론트엔드**: React, Vite, Next.js(SSR 및 SEO 고려)  
- **언어 및 파일 구조**: TypeScript를 사용하지 않으며, 모든 컴포넌트 및 페이지는 .jsx 파일로 작성합니다.  
- **데이터 소스**: 프로젝트 내 data/ 폴더의 CSV 파일  
- **데이터 저장소**: LocalStorage(이미지/코멘트 임시 저장), 향후 Firebase/백엔드 연동 가능  
- **스타일링**: Tailwind CSS(dark: 유틸리티 포함)  
- **기타**:  
  - CSV 파싱 및 렌더링 로직  
  - 이미지/코멘트는 LocalStorage에 임시 저장

# 개발 로드맵  
- **MVP 요구사항**:  
  - CSV 자동 로드 및 표 렌더링  
  - row 클릭 시 확장 디테일 뷰  
  - 이미지/코멘트 첨부 및 개수 표시  
  - 필터 기능  
  - 다크모드 지원  
  - 모든 컴포넌트는 .jsx 파일로 작성, TypeScript 미사용  
- **향후 개선**:  
  - 외부 DB(Firebase 등) 연동  
  - 첨부파일 클라우드 저장  
  - 고급 검색/필터링  
  - 사용자별 권한 관리

# 논리적 개발 순서  
- 1단계: CSV 데이터 파싱 및 표 렌더링  
- 2단계: row 클릭 시 확장 섹션 구현  
- 3단계: 이미지/코멘트 첨부 기능  
- 4단계: 필터 기능  
- 5단계: 다크모드 및 UI/UX 개선  
- 6단계: 데이터 저장소 확장(백엔드 연동 등)

# 위험 요소 및 대응 방안  
- **CSV 파싱 오류**: 다양한 CSV 포맷 지원, 예외처리 강화  
- **LocalStorage 용량 한계**: 첨부파일 크기 제한, 향후 클라우드 연동 고려  
- **UI/UX 복잡성**: 단계별 구현, 사용자 피드백 반영  
- **MVP 범위 초과 위험**: 핵심 기능 우선 개발, 부가 기능은 후순위로 미룸
